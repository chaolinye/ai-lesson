Lesson 2: GPTQ

量化不再是一次孤立事件——误差可以补偿

上节课我们做了朴素量化：对每个权重独立执行 round(x / scale)。这就像把一列数字一个一个四舍五入，完全不管它们之间的关系。

但考虑一个问题：

如果我要把 [3.2, 3.3, 3.4] 量化成整数。独立舍入是 [3, 3, 3]，总误差 0.9。
但如果我把 3.4 量化成 4，同时把 3.2 和 3.3 调整到 [3, 3]，总误差只有 0.5。
允许「串通」能得到更好的结果。

GPTQ (Generative Pre-trained Transformer Quantization) 的核心思想就是这个：量化一个权重后，调整剩下的权重来补偿误差。它在 2023 年由 Frantar 等人提出，是第一个把 LLM 压到 4-bit 而不明显损失精度的实用方法。

📌 核心 insight：朴素量化对所有权重一视同仁地做 round。GPTQ 意识到——权重之间不是独立的。你改变一个权重，输出会变，但你也可以通过改变其他权重来「弥补」这个变化。

从 OBS 到 OBQ 到 GPTQ

GPTQ 的思想来源于一个更古老的方法：Optimal Brain Surgeon (OBS)——它是一个剪枝（pruning）算法，问的问题是：「如果我要砍掉（设为 0）一个权重，我该怎么调整剩余权重来最小化损失？」

Optimal Brain Quantization (OBQ) 把剪枝换成量化：「如果我要把这个权重量化到最近的整数，我该怎么调整剩余权重来补偿？」

OBQ 的问题是——它太慢了。它一次处理一个权重，每处理完一个就要更新整个矩阵。对于一个 7B 模型来说，这是天文数字。

GPTQ 的三个关键加速：

  1. 一次性处理整列（列并行）：不是逐权重，而是逐列处理，同一列内的量化独立并行
  2. 懒惰批处理（Lazy Batch Update）：不是每量化一个权重就更新矩阵，而是攒一批一起更新，大幅减少内存访问
  3. Cholesky 预分解：Hessian 的逆矩阵预先算好，避免重复计算

Hessian 矩阵——量化指挥部的「敏感度地图」

GPTQ 的「秘密武器」是 Hessian 矩阵（记作 H）。

❓ 先问一个问题：一个权重矩阵 W，给一个输入 X，输出是 Y = WX。
现在我要量化 W。对于输出 Y 来说，哪些权重「更重要」？

答案取决于两件事：

  • 权重大小？ 不一定。一个大的权重可能对应一个总为零的输入通道。
  • 输入值？ 对了！权重要乘以输入才决定输出影响。

GPTQ 用校准数据集（100~200 条文本）跑一遍模型，记录每一层的输入 X，然后计算：

H = 2 × XTX

H 是一个 din × din 的矩阵。H[i][i] 表示「如果改变第 i 个权重，输出会变化多少」。H[i][j] 表示「改变第 i 个权重后，调整第 j 个权重能多大程度补偿」。

权重矩阵 W (d_out × d_in):              Hessian 矩阵 H (d_in × d_in):
┌ w₁₁  w₁₂  w₁₃  ... w₁ₙ ┐              ┌ h₁₁  h₁₂  h₁₃  ... h₁ₙ ┐
│ w₂₁  w₂₂  w₂₃  ... w₂ₙ │              │ h₂₁  h₂₂  h₂₃  ... h₂ₙ │
│  ⋮    ⋮    ⋮         ⋮  │              │  ⋮    ⋮    ⋮         ⋮  │
└ wₘ₁  wₘ₂  wₘ₃  ... wₘₙ ┘              └ hₙ₁  hₙ₂  hₙ₃  ... hₙₙ ┘

 逐列量化, 每列独立进行                H 告诉我们权重之间的「耦合关系」

GPTQ 的并行化策略：一次处理一列，而不是一个权重

GPTQ 算法（简化版）

对每一层：

  1. 收集数据：用校准数据集跑一遍，记录该层的输入 X
  2. 计算 H：H = 2 · XTX
  3. 计算 H-1（Cholesky 分解加速）：得到逆 Hessian
  4. 逐列量化并补偿：
    对权重矩阵的每一列 w：
    q = round(w / scale) ← 量化这一列
    δ = (w - q·scale) / [H⁻¹]qq · H⁻¹:,q ← 计算需要补偿的量
    剩下的列 -= δ ← 调整还没处理的权重来补偿误差
  5. 重复直到所有权重处理完
💡 直观理解：想象你在一个弹簧床上挖坑（量化一个权重），床会因此变形（输出误差）。你不是无视这个变形，而是按一下床的其他地方让床变回原来的形状（调整剩余权重）。Hessian 矩阵就是告诉你「按哪里有效」的说明书。

Group Size——精度的旋钮

GPTQ 的一个重要参数是 group size：

  • group_size = 128：每 128 个权重共享一个 scale。精度好，但存储 scale 的开销也大。
  • group_size = -1（等于整列）：整列共享一个 scale。省空间但精度差。
Group SizeScale 数量精度额外存储开销 (7B)
32很多最佳~128 MB
128适中很好~32 MB
-1 (整列)最少较差~1 MB
⚠️ 常见选择：group_size=128 是 GPTQ 的「黄金标准」——精度和存储开销的最佳平衡。几乎所有 TheBloke 发布的 GPTQ 模型都用 128。

动手环节：GPTQ 的误差补偿模拟

📝 练习：比较朴素量化 vs GPTQ 风格的误差补偿

下面的代码模拟一个 2D 权重矩阵的量化。我们比较两种策略：

  1. 朴素量化：独立 round 每个权重
  2. GPTQ 风格：量化一个后，用 Hessian 信息调整剩余
import torch
import numpy as np

def naive_quantize(w, bits=4):
    """独立舍入（上节课的做法）"""
    q_max = 2**(bits - 1) - 1
    scale = w.abs().max() / q_max
    q = torch.round(w / scale).clamp(-q_max, q_max)
    return q * scale  # 返回反量化后的值

def gptq_quantize_2d(w, bits=4):
    """
    简化版 GPTQ：用 Hessian 做误差补偿（2D 演示）
    注意：这是为教学简化的版本。真实 GPTQ 复杂得多。
    """
    q_max = 2**(bits - 1) - 1
    scale = w.abs().max() / q_max
    q = torch.zeros_like(w)
    
    # 人为构造一个简单的 Hessian 逆矩阵
    # 假设权重有相关性：H_inv[i,j] 表示 w[i] 和 w[j] 的耦合
    n = w.shape[0]
    H_inv = torch.eye(n) * 0.5 + 0.1  # 简单的耦合模拟
    
    w_cur = w.clone()
    for i in range(n):
        # 量化第 i 个权重
        q_i = torch.round(w_cur[i] / scale).clamp(-q_max, q_max)
        err = w_cur[i] - q_i * scale  # 量化误差
        
        q[i] = q_i
        
        # 补偿：用 H_inv 计算如何调整剩余权重
        if i < n - 1:
            compensation = err / H_inv[i, i] * H_inv[i, i+1:]
            w_cur[i+1:] -= compensation
    
    return q * scale

# 模拟一个强相关的小权重矩阵
torch.manual_seed(42)
W = torch.randn(16, 1) * 0.3 + 0.5  # 16 个权重，有正相关趋势

# 排序让它们更「耦合」
W = torch.sort(W.squeeze())[0].unsqueeze(1)

naive_result = naive_quantize(W)
gptq_result = gptq_quantize_2d(W)

naive_err = (W - naive_result).abs().mean().item()
gptq_err = (W - gptq_result).abs().mean().item()

print(f"原始权重:      {W.squeeze().tolist()}")
print(f"朴素量化重建:  {naive_result.squeeze().tolist()}")
print(f"GPTQ 风格重建: {gptq_result.squeeze().tolist()}")
print(f"\n朴素量化平均误差: {naive_err:.4f}")
print(f"GPTQ 风格平均误差: {gptq_err:.4f}")
print(f"改进:           {(naive_err - gptq_err) / naive_err * 100:.1f}%")
试试这个扩展：改变 H_inv 的构造方式——如果我把耦合系数设成 0（即假设所有权重独立），GPTQ 的补偿会退化到朴素量化吗？如果耦合放大（比如 0.5），补偿效果会更好还是更差？

GPTQ 的实际用法（代码一览）

你不会自己实现 GPTQ——有成熟的库帮你做。先有个印象就好，后续课程会动手实操：

# pip install gptqmodel transformers

from gptqmodel import GPTQModel, QuantizeConfig
from transformers import AutoTokenizer

model_id = "meta-llama/Meta-Llama-3-8B-Instruct"
quant_config = QuantizeConfig(bits=4, group_size=128)

model = GPTQModel.load(model_id, quant_config)
tokenizer = AutoTokenizer.from_pretrained(model_id)

# 校准——用 128 条文本决定量化参数
calibration_data = ["The answer to life is", "Machine learning is", ...]
model.quantize(calibration_data, batch_size=4)

model.save("llama3-8b-gptq")  # 保存量化后的模型

GPTQ 的优缺点

优点缺点
  • 精度高，4-bit 下 PPL 仅增加 0.3~0.5
  • 无需重训练
  • 支持 2/3/4/8 bit
  • Marlin kernel 推理极快
  • HuggingFace 生态完善
  • 需要校准数据集（麻烦）
  • 量化过程需要大量 CPU/GPU 内存
  • GPU only（没有 CPU 推理内核）
  • 对校准数据质量敏感（过拟合风险）
  • 量化比 AWQ 慢（需要算 Hessian）
