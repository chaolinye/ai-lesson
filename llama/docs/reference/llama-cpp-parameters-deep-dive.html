llama.cpp 核心参数原理速查

对应课程：Lesson 0004。官方依据：server README、cli README。

模型与计算参数

参数默认值控制什么常见调法
-m, --model—GGUF 模型路径。绝对路径，避免工作目录变化找不到。
-ngl, --n-gpu-layersautoGPU 中存放的模型层数；支持 auto/all/具体数字。显存够时设 all 或 999；OOM 时逐步降低。
-c, --ctx-size0 (=从模型加载)上下文窗口大小；决定 KV cache 上限。先 4096/8192；需要长上下文再调大，但要算显存。
-b, --batch-size2048逻辑最大 batch；影响 prompt prefill 吞吐量。长 prompt 处理慢时可适当增大；通常默认足够。
-ub, --ubatch-size512物理最大 batch；内存/显存峰值控制。OOM 于 prefill 阶段时可降低；通常不大于 -b。
-np, --parallel1并行 slot 数；server 并发请求容量。Agent 多并发时调高；总 KV cache 随 np 线性增长。
-n, --n-predict-1最大生成 token 数；-1 表示无限，-2 表示直到上下文填满。限制回复长度或防止无限生成。

采样参数（生成控制）

官方默认采样链：penalties;dry;top_n_sigma;top_k;typ_p;top_p;min_p;xtc;temperature。它们按顺序过滤/调整 logits，最后从剩余候选中采样。

参数默认值控制什么直觉
--temp0.8温度；对 logits 做缩放后采样。越低越确定（适合代码/事实），越高越发散（适合创意）。
--top-p0.9Nucleus sampling；只从累计概率达 p 的候选集中采样。过滤掉长尾低概率 token，保留核心候选。
--top-k40只保留概率最高的 k 个候选。硬截断；防止采样到明显离谱的 token。
--min-p0.05只保留概率 ≥ 最高概率 × min_p 的候选。相对截断；即使 top-p 很宽，也过滤掉极弱候选。
--presence-penalty0.0已出现过的 token 被惩罚，促进多样性。设为正值可减少重复；过高会破坏连贯性。
--frequency-penalty0.0按出现频率惩罚 token；频率越高惩罚越大。比 presence penalty 更激进地抑制重复。
-s, --seed-1随机种子；-1 表示每次随机。固定 seed 可复现输出；调试用。
