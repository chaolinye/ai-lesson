实操：完整链路从模型下载到 Agent 调用

今天不做新理论。我们把你已掌握的六课知识变成一次真实操作。

环境确认：你已有 llama-server b9611（CUDA 12.4 build）、RTX 5060 Ti 16GB、Python 3.13。没有 GGUF 模型文件——这正是我们要做的第一步。

操作计划

  1. 下载一个小型 GGUF 模型（测试用）
  2. 启动 llama-server 并验证 CUDA 工作
  3. 用 curl 测试 chat completion API
  4. 安装 openai Python 包，用 Python 调通
  5. 测试 JSON schema 约束输出
  6. 测试流式输出
  7. （可选）测试 function calling
  8. 关闭 server，为生产做准备

Step 1：下载第一个 GGUF 模型

你的 llama-server 支持 --hf-repo（-hf）直接从 Hugging Face 下载模型。我们先选一个小模型快速验证链路。

选择 A：Qwen2.5-1.5B-Instruct（推荐先试）

1.5B 参数，约 1GB，下载快、推理快，适合先跑通全链路。

mkdir C:\models
llama-server -hf Qwen/Qwen2.5-1.5B-Instruct-GGUF:Q4_K_M ^
  --host 127.0.0.1 --port 8080 ^
  -ngl all -c 4096

这会在启动时自动下载模型并开始服务。

选择 B：先下载再启动

如果你希望模型文件保存在本地，可以从 Hugging Face 手动下载 GGUF 文件，然后指定 -m 启动。例如：

  • 访问 Qwen2.5-1.5B-Instruct-GGUF
  • 下载 qwen2.5-1.5b-instruct-q4_k_m.gguf
  • 保存到 C:\models\
  • 用 -m C:\models\qwen2.5-1.5b-instruct-q4_k_m.gguf 启动

建议：先用选择 A（-hf）快速验证。全链路跑通后，再下载 7B/8B 模型用于实际 Agent。

Step 3：用 curl 测试 chat completion

没有设 --api-key 时：

curl http://127.0.0.1:8080/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -d "{\"model\":\"local\",\"messages\":[{\"role\":\"user\",\"content\":\"用一句话介绍 llama.cpp\"}],\"temperature\":0.7}"

设了 --api-key 时：

curl http://127.0.0.1:8080/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -H "Authorization: Bearer sk-llama-local-key" ^
  -d "{\"model\":\"local\",\"messages\":[{\"role\":\"user\",\"content\":\"用一句话介绍 llama.cpp\"}]}"

成功标志：返回 JSON，包含 choices[0].message.content 和 usage 字段。

Step 4：安装 openai 包并用 Python 调通

安装：

pip install openai

测试脚本：

from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:8080/v1",
    api_key="sk-llama-local-key",  # 如果没设 --api-key，随便填
)

response = client.chat.completions.create(
    model="local",
    messages=[{"role": "user", "content": "你好！你是谁？"}],
    temperature=0.7,
    max_tokens=200,
)

print(f"回复: {response.choices[0].message.content}")
print(f"Token 使用: {response.usage}")

Step 5：测试 JSON schema 约束输出

Python 代码：

from openai import OpenAI

client = OpenAI(base_url="http://127.0.0.1:8080/v1", api_key="sk-llama-local-key")

response = client.chat.completions.create(
    model="local",
    messages=[{"role": "user", "content": "列出三种编程语言及其创始年份"}],
    response_format={
        "type": "json_object",
        "schema": {
            "type": "object",
            "properties": {
                "languages": {
                    "type": "array",
                    "items": {
                        "type": "object",
                        "properties": {
                            "name": {"type": "string"},
                            "year": {"type": "integer"}
                        }
                    }
                }
            }
        }
    },
)

import json
data = json.loads(response.choices[0].message.content)
print(json.dumps(data, indent=2, ensure_ascii=False))
# 输出类似：
# {
#   "languages": [
#     {"name": "Python", "year": 1991},
#     {"name": "Java", "year": 1995},
#     {"name": "JavaScript", "year": 1995}
#   ]
# }

验证：JSON schema 保证输出一定是合法 JSON 且符合 schema 结构。如果模型输出非法 JSON，那是 llama.cpp 的 bug——因为它是在 grammar 级别约束的。

Step 6：测试流式输出

Python 流式：

from openai import OpenAI

client = OpenAI(base_url="http://127.0.0.1:8080/v1", api_key="sk-llama-local-key")

stream = client.chat.completions.create(
    model="local",
    messages=[{"role": "user", "content": "写一首关于 AI 的五言绝句"}],
    stream=True,
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)

Step 7（可选）：测试 function calling

Python 代码：

from openai import OpenAI

client = OpenAI(base_url="http://127.0.0.1:8080/v1", api_key="sk-llama-local-key")

response = client.chat.completions.create(
    model="local",
    messages=[{"role": "user", "content": "北京的天气怎么样？"}],
    tools=[{
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {"type": "string"}
                },
                "required": ["city"]
            }
        }
    }],
    tool_choice="auto",
)

msg = response.choices[0].message
if msg.tool_calls:
    for tc in msg.tool_calls:
        print(f"调用工具: {tc.function.name}")
        print(f"参数: {tc.function.arguments}")
else:
    print(f"回复: {msg.content}")

Step 8：关闭 server

在 server 终端按 Ctrl+C。确认 nvidia-smi 显示显存已释放。

生产准备建议

  • 从 1.5B 测试模型切换到你真正需要的 7B/8B 模型。
  • 选择 Q5_K_M 或 Q4_K_M 量化。
  • 设 --api-key（如果暴露到网络）。
  • 根据 Agent 并发调整 -np。
  • 建议写一个 PowerShell 脚本保存启动命令，避免每次输一长串。
