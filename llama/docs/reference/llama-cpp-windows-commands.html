llama.cpp Windows 命令速查

配套课程：Lesson 0002。官方依据：install.md、build.md、server README。

路线 A：预构建安装

winget install llama.cpp
llama-server --help

注意：Winget 安装快，但具体是否包含你想要的 CUDA 后端，要以 llama-server --help、启动日志、实际 GPU 使用情况为准。若不符合预期，走路线 B 自行 CUDA 构建。

路线 B：源码 CUDA 构建

git clone https://github.com/ggml-org/llama.cpp
cd llama.cpp
cmake -B build -DGGML_CUDA=ON
cmake --build build --config Release -j

Windows 上按官方 build docs，通常需要 Visual Studio 2022 C++ 桌面开发组件、CMake、Git，以及 NVIDIA CUDA Toolkit。请在 “Developer PowerShell/Command Prompt for VS 2022” 中构建。

启动 server

llama-server \
  -m C:\models\your-model.gguf \
  --host 127.0.0.1 \
  --port 8080 \
  -ngl auto \
  -c 4096

测试 OpenAI-compatible API

curl http://127.0.0.1:8080/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -d "{\"model\":\"local\",\"messages\":[{\"role\":\"user\",\"content\":\"Say hello in Chinese.\"}],\"temperature\":0.7}"

参数速查

参数作用初始建议
-m模型 GGUF 路径。先用 7B/8B instruct GGUF。
--host监听地址。本机 Agent 用 127.0.0.1；局域网访问再考虑 0.0.0.0。
--portHTTP 端口。8080。
-nglGPU offload 层数；新版本可用 auto/all。先 auto 或 all；OOM 再降。
-c上下文大小；影响 KV cache。先 4096 或 8192。
-b逻辑 batch；影响 prompt processing。默认即可，遇到长 prompt 性能问题再调。
-ub物理 micro-batch；控制显存峰值。默认即可，OOM 可降。
-np并行 slot 数。先 1；Agent 多并发再调。
