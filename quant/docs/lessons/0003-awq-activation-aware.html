Lesson 3: AWQ

不补偿，而是保护——激活值感知量化

你已经学了两条路：

  • 朴素量化：所有权重一视同仁，独立舍入
  • GPTQ：量化后补偿，用 Hessian 矩阵告诉你怎么补

AWQ 走的是第三条路。它来自 MIT HAN Lab（Song Han 组），发表在 MLSys 2024 并获得最佳论文。它的 insight 非常直观：

只有 ~1% 的权重通道是「重要的」——那些和大的激活值相连的通道。
保护这 1%，剩下的可以放心大胆地量化。

一个类比：摄影师的取舍

想象你有一张 4K 照片，但手机存储空间不够了，需要压缩到 1MB。你怎么办？

  • 朴素量化：整张照片统一降低质量——人脸和背景一样糊
  • GPTQ：先压缩一部分，然后调整周围的像素来「弥补」
  • AWQ：识别出照片中的人脸（重要区域），保留人脸细节，背景狠狠压缩
📌 AWQ 的核心 insight：LLM 的权重矩阵中有少量「salient channels」——它们的激活值特别大，量化误差会被放大。如果只保护这些通道，其余通道可以大胆量化，整体效果损失极小。

OUTLIER 现象——为什么 LLM 这么特殊

2022 年，Dettmers 等人发现了一个令人惊讶的现象：LLM 的激活值中存在巨大的 outlier 特征。

在一层 attention 中，某些 hidden dimension 的激活值可以达到其他维度的 10~100 倍。8-bit 量化下，这些 outlier 会导致巨大的误差。

Hidden dimension:   1   2   3   4   5   6   7  ...
激活值 (normalized): 0.1 0.3 0.2 12.8 0.1 0.4 9.5 ...
                             ↑                 ↑
                          outlier!          outlier!

这些 outlier 不是随机的——它们在同一个 hidden dimension 上持续出现，也就是说，某些通道就是「天生重要」。

AWQ 的核心算法

AWQ 的算法出奇地简单，分为三步：

Step 1: 识别重要通道

用少量校准数据（~128 条）跑一遍模型，记录每个线性层的输入激活值 X。对每个输出通道 i，计算其激活值的平均大小：

si = mean(|Xi|)

si 越大，第 i 个通道越重要。

Step 2: 缩放后再量化

对重要通道的权重，在做 scale 之前先乘一个缩放因子 α（经验值 α ≈ 0.5~1.0）：

如果 si 是 top 1%，则 Wi ← Wi / α
然后正常做量化：q = round(Wi / scale_i)

「除以 α」让重要权重的数值变小，在固定 bit 范围内获得更精细的表示。

Step 3: 将缩放吸收到上一层

这一步是 AWQ 的妙笔——推理时不能多一次乘 α 操作。AWQ 把 α 吸收到上一层的权重中：

设当前层权重 W，上一层权重 Wprev，输入 X
量化：Y = Q(W) · (α · X) = [Q(W) · α] · X （把 α 吸收到量化后的权重里）
等价于：Y = Wprev 中对应的列 × α，然后正常做量化

💡 为什么这很聪明？因为 α 被吸收后，推理时没有任何额外计算。权重已经量化好了，而这个「保护重要通道」的操作被免费地编码进了量化权重里。相比 GPTQ 的逐列补偿循环，AWQ 只需要一次校准前向传播——快~4 倍。

AWQ 与 GPTQ 的哲学对比

GPTQ

「我量化了一个权重，造成了误差。让我调整其他权重来弥补。」

📊 需要 Hessian 矩阵
⚙️ 逐列循环 + 补偿
⏱️ 量化慢（算 H 耗时）

AWQ

「这个通道的权重很重要（因为激活值大）。让我在量化前保护它。」

📊 只需要激活值统计
⚙️ 一次前向传播就够
⏱️ 量化快 ~4×

为什么 AWQ 效果更好？

在 4-bit 量化下，AWQ 通常比 GPTQ 的 PPL 低 0.1~0.2（越小越好）。原因在于：

  1. 激活值意识：GPTQ 只看权重本身和 Hessian（基于输入数据的统计），而 AWQ 直接看激活值大小——这才是真正决定输出影响力的因素
  2. 没有过拟合风险： GPTQ 的 Hessian 拟合可能过度适应校准数据；AWQ 只统计激活值幅度，更鲁棒
  3. Uniform 存储格式：所有权重仍然是 INT4，没有混合精度，硬件友好

动手环节：模拟 AWQ 的 Per-Channel Scaling

📝 练习：手动实现 AWQ 风格的保护性缩放

模拟一个场景：一个权重矩阵中少数通道有大的激活值，对比「直接量化」和「先缩放保护再量化」的效果。

import torch
import math

def awq_protect_quantize(weight, activation_scale, alpha=0.5, bits=4):
    """
    模拟 AWQ 的 per-channel scaling 保护性量化
    
    weight: (d_out, d_in) 权重矩阵
    activation_scale: (d_out,) 每个输出通道的激活值大小
    alpha: 缩放系数（论文建议 0.5~1.0）
    """
    q_max = 2**(bits - 1) - 1
    
    # Step 1: 找出重要的通道（top 1%）
    threshold = torch.quantile(activation_scale, 0.99)
    important = activation_scale >= threshold
    print(f"重要通道: {important.sum().item()} / {important.numel()} ({important.sum().item()/important.numel()*100:.1f}%)")
    
    # Step 2: 对重要通道做保护性缩放
    weight_scaled = weight.clone()
    weight_scaled[:, important] = weight[:, important] / alpha
    
    # Step 3: 量化（per-channel scale）
    result = torch.zeros_like(weight)
    for i in range(weight.shape[1]):  # 每个输出通道独立量化
        w = weight_scaled[:, i] if important[i] else weight[:, i]
        scale = w.abs().max() / q_max
        q = torch.round(w / scale).clamp(-q_max, q_max)
        result[:, i] = q * scale
    
    return result

def naive_channel_quantize(weight, bits=4):
    """朴素 per-channel 量化（没有保护性缩放）"""
    q_max = 2**(bits - 1) - 1
    result = torch.zeros_like(weight)
    for i in range(weight.shape[1]):
        w = weight[:, i]
        scale = w.abs().max() / q_max
        q = torch.round(w / scale).clamp(-q_max, q_max)
        result[:, i] = q * scale
    return result

# 构造模拟数据
torch.manual_seed(42)
d_out, d_in = 64, 128
weight = torch.randn(d_out, d_in) * 0.1

# 模拟激活值：大部分小，1% 的几个通道特别大
activation_scale = torch.ones(d_in) * 0.1
activation_scale[torch.randperm(d_in)[:2]] = 10.0  # 2 个 outlier 通道

print("=" * 50)
print("测试 AWQ 风格保护量化 vs 朴素 channel-wise 量化")
print("=" * 50)

naive_q = naive_channel_quantize(weight)
awq_q = awq_protect_quantize(weight, activation_scale, alpha=0.5)

naive_mse = ((weight - naive_q) ** 2).mean().item()
awq_mse = ((weight - awq_q) ** 2).mean().item()

print(f"\n朴素 per-channel 量化 MSE: {naive_mse:.6f}")
print(f"AWQ 风格保护量化 MSE:     {awq_mse:.6f}")
print(f"改进:                     {(naive_mse - awq_mse) / naive_mse * 100:.1f}%")

# 只看重要通道
print(f"\n对 2 个重要通道单独看:")
for idx in torch.where(activation_scale >= activation_scale.quantile(0.99))[0]:
    col = weight[:, idx]
    naive_err = (col - naive_q[:, idx]).abs().mean().item()
    awq_err = (col - awq_q[:, idx]).abs().mean().item()
    print(f"  通道 {idx.item():3d}: 朴素误差={naive_err:.4f}, AWQ误差={awq_err:.4f}, 改进={(naive_err-awq_err)/naive_err*100:.1f}%")
试试这些扩展：
• 把 alpha 从 0.5 改成 0.1、0.3、0.9——观察效果变化。alpha 太小会怎样？
• 把 outlier 通道从 2 个增加到 10 个——保护 1% 的假设还成立吗？

AWQ 的实际用法

# pip install autoawq transformers

from awq import AutoAWQForCausalLM
from transformers import AutoTokenizer

model_id = "meta-llama/Meta-Llama-3-8B-Instruct"

model = AutoAWQForCausalLM.from_pretrained(model_id)
tokenizer = AutoTokenizer.from_pretrained(model_id)

# 配一个简单的校准数据集
calibration_data = [
    "The meaning of life is",
    "Machine learning is a subset of",
    "In the beginning, there was",
]

# 量化——一行搞定
model.quantize(
    tokenizer,
    quant_config={"bits": 4, "group_size": 128, "zero_point": True},
    calib_data=calibration_data
)

# 保存
model.save_quantized("llama3-8b-awq")
tokenizer.save_pretrained("llama3-8b-awq")
