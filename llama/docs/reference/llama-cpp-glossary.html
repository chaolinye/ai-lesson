llama.cpp 术语速查

这是后续课程共用的术语表。每次遇到参数时，优先把它放回这些概念里理解。

核心对象

llama.cpp
一个 C/C++ LLM 推理引擎，重点是本地运行 GGUF 模型，并支持 CPU、CUDA、Metal、Vulkan 等后端。官方 README 称其支持量化模型、GPU/CPU 混合推理和 OpenAI-compatible server。
GGUF
llama.cpp 当前主要使用的模型文件格式。它把模型权重、tokenizer、模型元数据等放在一个文件中。运行时常用 -m model.gguf 指定。
量化 / Quantization
用更少 bit 表示权重的编码方式。目标通常是降低内存/显存占用、提高吞吐，但会带来不同程度质量损失。常见如 Q4_K_M、Q5_K_M、Q8_0。
模型权重 / Weights
训练后固定下来的参数。加载模型时主要占用内存或显存。量化主要压缩的是权重。
KV cache
推理时为注意力机制保存的 key/value 中间状态。它随上下文长度、层数、隐藏维度、并发数增长；不是模型文件的一部分，但会显著占用内存/显存。

推理阶段

Prefill / Prompt processing
把已有 prompt 一次性喂进模型，建立 KV cache。通常更像矩阵-矩阵计算，受 batch 参数影响明显。
Decode / Token generation
逐 token 生成回复。每一步用当前 token 和已缓存的 KV 计算下一个 token。用户感受到的 tokens/s 多指 decode 速度。
Context size
一次会话最多能容纳的 token 数，包括系统提示、用户消息、历史、模型输出。llama.cpp 常用 -c 或 --ctx-size 设置。

常见命令参数

-m / --model
GGUF 模型路径。
-ngl / --n-gpu-layers
把多少层 offload 到 GPU。显存足够时常设很大，让尽可能多的层在 GPU 上运行。
-c / --ctx-size
上下文窗口大小。越大，KV cache 占用越高。
-b / --batch-size
逻辑 batch 大小，影响 prompt processing。
-ub / --ubatch-size
物理 micro-batch 大小，用于在内存限制内切分计算。
-t / --threads
CPU 线程数。GPU offload 多时影响相对变小，但仍会影响采样、未 offload 层等。
--temp
temperature，控制采样随机性。越低越确定，越高越发散。
--top-p
nucleus sampling，只在累计概率达到 p 的候选 token 集中采样。
-np / --parallel
server 中的并行 slot 数。并发越高，总 KV cache 需求越高。
