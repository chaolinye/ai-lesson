Lesson 5: 动手量化一个真正的模型

用 llama.cpp 从下载到量化到推理——完整走一遍

前四节课全是理论。这节课——动手。

你会：

  1. 下载一个真正的模型（SmolLM2-135M，~270MB，非常轻量）
  2. 用 llama-quantize 把它量化成 Q4_K_M
  3. 对比量化前后的文件大小
  4. 用 llama-cli 跑推理，看看量化后模型表现如何
  5. 额外：试试多个量化类型，对比精度和大小
⚡ 你的装备：RTX 5060 Ti (16GB) + llama.cpp (CUDA) + Python 3.13
这是本地部署的「黄金配置」——这节实战课为你量身设计。

Step 0：准备工作

打开终端（PowerShell 或 CMD，不是 bash）：

# 设置 llama.cpp 路径（根据你的安装位置）
$env:LLAMA_PATH = "D:\Application\llama-b9611-bin-win-cuda-12.4-x64"

# 创建工作目录
mkdir D:\Code\learn\quant2\models -Force
cd D:\Code\learn\quant2\models

# 验证 llama-quantize 可用
& "$env:LLAMA_PATH\llama-quantize.exe" --help
💡 如果报错：确认路径里的版本号（b9611）是否正确。你可以用 ls D:\Application\llama-* 查看准确路径。

Step 1：下载一个模型（F16 GGUF）

我们从 HuggingFace 下载 SmolLM2-135M 的 F16 版本。这是模型的最小可用精度——未量化，用来做量化的「原材料」。

# 用 huggingface-cli 下载（Python 自带 huggingface_hub）
python -m huggingface_hub download mradermacher/SmolLM2-135M-GGUF SmolLM2-135M.f16.gguf --local-dir .

# 查看下载后的文件
ls *.gguf | Select-Object Name, @{N="SizeMB";E={[math]::Round($_.Length/1MB, 1)}}

# 你应该看到类似：
# SmolLM2-135M.f16.gguf    270.5 MB
⚠️ 如果下载慢：可以用镜像站 --endpoint https://hf-mirror.com。
或者直接用浏览器访问 这个链接 下载。

Step 2：量化！

用 llama-quantize 把 F16 模型量化成 Q4_K_M：

# 量化到 Q4_K_M
& "$env:LLAMA_PATH\llama-quantize.exe" `
    SmolLM2-135M.f16.gguf `
    SmolLM2-135M-Q4_K_M.gguf `
    Q4_K_M

# 输出应该类似：
# main: quantizing 'SmolLM2-135M.f16.gguf' to 'SmolLM2-135M-Q4_K_M.gguf' as Q4_K_M
# main: quantize from 16 to 4.50 bits = 71.9% compression
# main: model size = 270.48 MB → 89.75 MB

看到了吗？270 MB → 90 MB，压缩了 71.9%。这就是量化！

Step 3：对比所有量化类型

如果你有耐心，可以一次性生成多个量化版本做对比：

# 尝试多种量化类型
$types = @("Q2_K", "Q3_K_M", "Q4_K_M", "Q5_K_M", "Q6_K", "Q8_0")

foreach ($type in $types) {
    $output = "SmolLM2-135M-$type.gguf"
    Write-Host "Quantizing to $type..."
    & "$env:LLAMA_PATH\llama-quantize.exe" SmolLM2-135M.f16.gguf $output $type 2>&1 | Out-Null
}

# 查看所有文件大小
Get-ChildItem *.gguf | Select-Object Name, @{N="SizeMB";E={[math]::Round($_.Length/1MB, 1)}} | Sort-Object SizeMB

你会看到这样的对比：

Name                          SizeMB
----                          ------
SmolLM2-135M-Q2_K.gguf         53.8     ← Q2: 极低精度，极小
SmolLM2-135M-Q3_K_M.gguf       67.4     ← Q3: 一般
SmolLM2-135M-Q4_K_M.gguf       89.8     ← ✅ 推荐: 最佳性价比
SmolLM2-135M-Q5_K_M.gguf      107.8     ← Q5: 更高精度
SmolLM2-135M-Q6_K.gguf        125.9     ← Q6: 接近无损
SmolLM2-135M-Q8_0.gguf        144.1     ← Q8: 几乎无损
SmolLM2-135M.f16.gguf         270.5     ← F16: 原始精度
🔍 自己发现规律：
从 F16 (270 MB) 到 Q4_K_M (90 MB) = 压缩 66%。
从 Q4_K_M 到 Q5_K_M (108 MB) = 只多了 18 MB，但精度提升明显。
从 Q5_K_M 到 Q8_0 (144 MB) = 多 36 MB，但提升越来越小。
边际效益递减——这就是为什么 Q4_K_M 是黄金选择。

Step 4：跑推理验证

用 llama-cli 测试量化后的模型：

# 用 Q4_K_M 版本推理
& "$env:LLAMA_PATH\llama-cli.exe" `
    -m SmolLM2-135M-Q4_K_M.gguf `
    -p "The meaning of life is" `
    -n 50 `
    -t 8

# 解释参数：
# -m  模型文件
# -p  prompt
# -n  生成多少个 token
# -t  线程数

你应该能看到模型续写了一段文字。试试不同 prompt：

& "$env:LLAMA_PATH\llama-cli.exe" -m SmolLM2-135M-Q4_K_M.gguf -p "Machine learning is" -n 50 -t 8

Step 5：对比不同精度的输出

这是最有趣的部分——比较 Q2_K 和 Q8_0 的输出质量：

# Q2_K（最低精度）
& "$env:LLAMA_PATH\llama-cli.exe" -m SmolLM2-135M-Q2_K.gguf -p "The capital of France is" -n 30 -t 8

# Q4_K_M（推荐）
& "$env:LLAMA_PATH\llama-cli.exe" -m SmolLM2-135M-Q4_K_M.gguf -p "The capital of France is" -n 30 -t 8

# Q8_0（最高精度）
& "$env:LLAMA_PATH\llama-cli.exe" -m SmolLM2-135M-Q8_0.gguf -p "The capital of France is" -n 30 -t 8
⚠️ 小模型的限制：SmolLM2-135M 本身只有 1.35 亿参数，能力有限（比 GPT-2 还小）。不同量化类型之间的差异可能不明显。但流程是一样的——你学会了如何对任何模型做量化。

Step 6：用 GPU 加速推理

你有 RTX 5060 Ti（16GB 显存）！让 llama-cli 用 GPU 跑：

# 指定 GPU 层数（-ngl 表示 offload 到 GPU 的层数）
# SmolLM2-135M 大概有 12 层，全部 offload：
& "$env:LLAMA_PATH\llama-cli.exe" `
    -m SmolLM2-135M-Q4_K_M.gguf `
    -p "Once upon a time" `
    -n 50 `
    -t 8 `
    -ngl 99          # 所有层都放到 GPU

# 用 llama-bench 测速
& "$env:LLAMA_PATH\llama-bench.exe" -m SmolLM2-135M-Q4_K_M.gguf -ngl 99

Bonus：如果装了 PyTorch——bitsandbytes 即时量化

如果未来你装好了 PyTorch + CUDA，可以试试这个一行代码的量化方法：

# 先安装
pip install transformers bitsandbytes accelerate

# 然后——一行代码实现 4-bit 量化推理
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig
import torch

quant_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",      # NormalFloat4 (QLoRA 方案)
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,  # 双重量化，再省一点
)

model = AutoModelForCausalLM.from_pretrained(
    "HuggingFaceTB/SmolLM2-135M",
    quantization_config=quant_config,
    device_map="auto",
)
tokenizer = AutoTokenizer.from_pretrained("HuggingFaceTB/SmolLM2-135M")

# 推理
inputs = tokenizer("The meaning of life is", return_tensors="pt").to("cuda")
outputs = model.generate(**inputs, max_new_tokens=50)
print(tokenizer.decode(outputs[0]))
💡 bitsandbytes 的特点：不需要提前量化！下载原始 FP16 模型，在加载时即时量化为 NF4。特别适合快速测试和 QLoRA 微调。缺点是推理速度不如 GGUF 或 AWQ。

完整的本地部署决策树

你的目标是本地部署一个 LLM？

1. 选模型
   ├── 你的显存够大 (≥16GB) → 7B 或 13B 模型，Q4_K_M
   ├── 显存中等 (8~12GB) → 7B 模型，Q4_K_M
   └── 显存小 (≤6GB) → 3B 或 1.5B 模型

2. 选工具
   ├── 简单快捷 → Ollama（一行命令拉模型）
   ├── 最大控制 → llama.cpp CLI
   ├── 需要 API → llama-server（已经在跑了！）
   └── 需要微调 → bitsandbytes + QLoRA

3. 选量化类型
   ├── 不知道怎么选 → Q4_K_M（永远不后悔）
   ├── 显存吃紧 → Q4_K_S 或 Q3_K_M
   ├── 质量优先 → Q5_K_M 或 Q6_K
   └── 只是测试 → Q2_K

你的 RTX 5060 Ti (16GB) 可以轻松跑：
  - 7B 模型 Q4_K_M  (~4.1 GB)
  - 7B 模型 Q8_0     (~7.2 GB)  ← 接近无损
  - 13B 模型 Q4_K_M  (~7.5 GB)
  - 甚至 70B Q2_K    (~27 GB, 需要内存卸载一部分)
