llama.cpp 显存优化速查

基于官方文档与社区实践。目标：在 16GB 显存上最大化模型大小、上下文长度、并发数。

显存占用公式（回顾）

总 VRAM = 量化后权重 + KV cache × 并发数 + 计算缓冲 + 系统/驱动开销

优化技术一览

技术参数省什么省多少代价
Flash Attention-fa onKV cache 显存显著（长上下文尤其明显）极小（新版本默认启用）
KV cache 量化-ctk q8_0 -ctv q8_0KV cache 显存f16→q8_0 省 50%轻微精度损失
KV cache 量化（激进）-ctk q4_0 -ctv q4_0KV cache 显存f16→q4_0 省 75%更大精度损失
减少 GPU offload-ngl 20（而非 all）权重显存部分层移回 CPU速度下降
降低 ubatch-ub 256计算缓冲减少峰值显存prefill 变慢
降低并发-np 1KV cache 倍率×1 而非 ×N一次只能处理一个请求
更低量化选 Q4_K_M 而非 Q5_K_M权重约 0.7GB (7B)质量损失
禁用 mmap--no-mmap减少 pageout不定加载更慢

场景策略

场景 1：8B Q5_K_M，8K 上下文，单并发
权重 ~5.5 GB + KV cache f16 ~4 GB + 余量 ~2 GB = ~11.5 GB → ✅ 舒适
场景 2：8B Q5_K_M，32K 上下文，单并发
权重 ~5.5 GB + KV cache f16 ~16 GB = ~21.5 GB → ❌ 超显存
方案：-ctk q8_0 -ctv q8_0 → KV cache ~8 GB → 总 ~15.5 GB → ✅ 有风险，建议再降 -ngl
场景 3：14B Q4_K_M，8K 上下文，单并发
权重 ~8.2 GB + KV cache f16 ~7 GB + 余量 ~2 GB = ~17.2 GB → ⚠️ 略超
方案：-fa on + -ctk q8_0 -ctv q8_0 → KV cache ~3.5 GB → 总 ~13.7 GB → ✅
场景 4：8B Q4_K_M，8K 上下文，4 并发
权重 ~4.1 GB + KV cache f16 ×4 ~16 GB → ❌ 超显存
方案：-ctk q8_0 -ctv q8_0 → KV cache ×4 ~8 GB → 总 ~14.1 GB → ⚠️ 有风险

相关参数速查

参数作用
-fa onFlash Attention。节省注意力计算显存。新版本默认 auto。
-ctk TYPEK cache 数据类型：f32 / f16 / bf16 / q8_0 / q4_0 等。默认 f16。
-ctv TYPEV cache 数据类型。同上。
-kvo / -nkvoKV cache 是否 offload 到 GPU。默认开启。
-ngl NGPU offload 层数。降低可省显存但降速。
-ub N物理 micro-batch。OOM 安全阀。
-np N并行 slot 数。每个 slot 独立 KV cache。
-cb / -nocbcontinuous batching。多请求时提高 GPU 利用率。
--context-shift上下文转移。超出上下文时滑动窗口，避免重新计算。
--no-mmap关闭内存映射。加载更慢但减少 pageout。
