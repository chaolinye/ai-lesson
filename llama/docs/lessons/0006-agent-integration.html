对接 Agent：OpenAI API、JSON schema、Function calling

今天的目标：让你的 llama.cpp server 不只是"能跑模型"，而是成为 Agent 可调用的标准后端服务。

与你的任务的关系：你学 llama.cpp 的最终目的，是让 Agent 通过 API 调本地模型。前五课已经覆盖了安装、量化、参数、显存。这节课把最后一块拼图拼上——让 Agent 能发消息、约束输出格式、甚至调用工具。

1. llama.cpp server 暴露的 API 全景

官方 server README 列出了所有端点。核心是：

端点作用你的 Agent 需要哪个？
POST /v1/chat/completions对话生成✅ 最主要
POST /v1/completions文本补全视情况
POST /v1/embeddings向量嵌入如果 Agent 需要 RAG
GET /v1/models获取模型信息客户端初始化验证
GET /slots查看并行 slot 状态调试用
GET /metricsPrometheus 指标监控（需 --metrics）

最重要的是：这些端点和 OpenAI 的 API 格式一致。这意味着任何兼容 OpenAI API 的客户端库（如 OpenAI Python SDK）可以直接连你的本地 server，只需改 base_url。

2. 启动带认证的 server

一个完整的启动命令：

llama-server ^
  -m C:\models\my-8b-instruct.Q5_K_M.gguf ^
  --host 127.0.0.1 --port 8080 ^
  -ngl all -c 8192 -fa on ^
  -np 1 --temp 0.7 ^
  --api-key "sk-llama-local-key"

官方 server README 说 --api-key 可以设多个逗号分隔的值，也可以用 --api-key-file 指定文件。不设 api-key 时，所有请求都可以访问——本地开发可以，暴露到网络时必须设。

3. 用 curl 测试 chat completion

curl http://127.0.0.1:8080/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -H "Authorization: Bearer sk-llama-local-key" ^
  -d "{\"model\":\"local\",\"messages\":[{\"role\":\"user\",\"content\":\"用中文介绍 llama.cpp\"}],\"temperature\":0.7}"

返回的 JSON 结构：

{
  "id": "chat-xxx",
  "object": "chat.completion",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "llama.cpp 是一个...（模型生成的回复）"
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 42,
    "completion_tokens": 128,
    "total_tokens": 170
  }
}

4. 流式输出（streaming）

Agent 场景中，流式输出让用户更快看到第一个 token。只需在请求体加 "stream": true：

curl http://127.0.0.1:8080/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -H "Authorization: Bearer sk-llama-local-key" ^
  -d "{\"model\":\"local\",\"messages\":[{\"role\":\"user\",\"content\":\"数到 5\"}],\"stream\":true}"

返回 SSE (Server-Sent Events) 格式，每个 chunk 包含 delta 内容。

5. JSON schema 约束输出格式

当 Agent 需要模型输出结构化数据（如 JSON）时，可以用 response_format 约束。官方 grammars README 说明 server 支持将 JSON schema 自动转换为 GBNF grammar，在生成时约束 token 选择。

curl http://127.0.0.1:8080/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -H "Authorization: Bearer sk-llama-local-key" ^
  -d "{
    \"model\":\"local\",
    \"messages\":[{\"role\":\"user\",\"content\":\"列出三个编程语言及其创始年份\"}],
    \"response_format\":{
      \"type\":\"json_object\",
      \"schema\":{
        \"type\":\"object\",
        \"properties\":{
          \"languages\":{
            \"type\":\"array\",
            \"items\":{
              \"type\":\"object\",
              \"properties\":{
                \"name\":{\"type\":\"string\"},
                \"year\":{\"type\":\"integer\"}
              }
            }
          }
        }
      }
    }
  }"

原理：llama.cpp 在收到请求时把 JSON schema 编译成 GBNF grammar，然后在采样时只允许符合 grammar 的 token。这意味着模型不会输出非法 JSON。

关键限制：JSON schema 约束的是"输出格式"，不是"输出内容"。模型仍然可能在 schema 框架内输出错误的事实——JSON schema 保证语法正确，不保证语义正确。

6. Function calling（工具调用）

官方 function-calling.md 说明：llama.cpp server 支持 OpenAI 风格的 function calling。你定义 tools，模型判断是否调用以及用什么参数。

curl http://127.0.0.1:8080/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -H "Authorization: Bearer sk-llama-local-key" ^
  -d "{
    \"model\":\"local\",
    \"messages\":[{\"role\":\"user\",\"content\":\"北京的天气怎么样？\"}],
    \"tools\":[{
      \"type\":\"function\",
      \"function\":{
        \"name\":\"get_weather\",
        \"description\":\"获取指定城市的天气信息\",
        \"parameters\":{
          \"type\":\"object\",
          \"properties\":{
            \"city\":{\"type\":\"string\"}
          },
          \"required\":[\"city\"]
        }
      }
    }],
    \"tool_choice\":\"auto\"
  }"

模型可能返回：

{
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": null,
      "tool_calls": [{
        "type": "function",
        "function": {
          "name": "get_weather",
          "arguments": "{\"city\":\"北京\"}"
        }
      }]
    }
  }]
}

工作流：

  1. Agent 发请求时携带 tools 定义。
  2. 模型判断需要调用 get_weather，返回 tool_calls。
  3. Agent 在本地执行工具函数（如查天气 API）。
  4. Agent 把工具结果作为新的 role:tool 消息发给模型。
  5. 模型基于工具结果生成最终回复。

7. 用 Python 客户端连接

from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:8080/v1",
    api_key="sk-llama-local-key",
)

response = client.chat.completions.create(
    model="local",
    messages=[{"role": "user", "content": "你好！"}],
    temperature=0.7,
    stream=False,
)
print(response.choices[0].message.content)

关键是 base_url 指向你本地 server 的 /v1 路径。其他用法和 OpenAI SDK 完全一致。

8. 并发：Agent 多请求场景

你的 Agent 可能同时处理多个任务。这时需要：

  • -np N：N 个并行 slot。每个 slot 独立处理一个请求。
  • continuous batching 默认开启，动态混合多个请求的 token 计算。
  • 每个 slot 有独立的 KV cache。总显存 = 权重 + KV cache × N。

建议：从 -np 1 开始跑通链路，再逐步增加到 2、4，观察显存和响应时间。如果 Agent 的请求不密集，-np 1 配合 continuous batching 的排队机制也够用。
