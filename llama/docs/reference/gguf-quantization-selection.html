GGUF 量化选择速查

基于 llama.cpp 官方数据与社区共识，服务于 5060 Ti 16GB 等桌面级环境。

量化名称解码

以 Q5_K_M 为例：

  • Q5：每权重约 5 bit。
  • K：K-quant（improved quantization），通常比旧版更精细。
  • M/S/L：混合精度档位。M = Medium，S = Small（更省），L = Large（更保真）。

7B 级别量化对比（官方 perplexity 近似值）

注意：Perplexity 只在同模型/同 tokenizer 内部可比，不能跨模型直接比较数值大小。官方 perplexity 工具 主要用于评估量化损失。

量化约大小(7B)相对 FP16 ppl 变化质量评价适合场景
FP1613.0 GB—基准参考值，通常不直接跑。
Q8_07.0 GB+0.03%近乎无损小模型、质量敏感、显存充足。
Q6_K5.5 GB+0.13%极好高质量/体积最佳折中。
Q5_K_M4.8 GB+0.39%很好16GB 显存首选候选之一。
Q4_K_M4.1 GB+1.68%好常用默认推荐；显存紧张或跑更大模型时。
Q4_K_S3.9 GB+2.62%可接受极限省空间，但可能牺牲推理质量。

5060 Ti 16GB 显存选择策略

推理时总 VRAM 占用 ≈ 量化后权重 + KV cache + 计算缓冲 + server 开销。

  • 7B/8B 模型：Q5_K_M 通常舒适；Q4_K_M 可为长上下文留更多余量。
  • 14B 模型：Q5_K_M ≈ 9.6 GB；Q4_K_M ≈ 8.2 GB。若上下文长或并发高，优先 Q4_K_M。
  • 22B/27B 模型：Q4_K_M 约 12–14 GB，已接近上限。上下文和并发要保守。
  • 32B+：即使 Q4_K_M 也可能超过显存，需考虑 CPU+GPU 混合或更小量化。

找模型

  • Hugging Face 搜索：?apps=llama.cpp
  • 模型卡要确认：Instruct / chat template / context length / license。
  • llama-server 可直接下载：llama-server -hf user/model:Q5_K_M
